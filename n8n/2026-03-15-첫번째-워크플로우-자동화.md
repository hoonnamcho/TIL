### Context
n8n을 사용하여 첫 번째 워크플로우 자동화를 구축했습니다. 이 워크플로우는 디스코드에서 메시지를 수신하면 트리거가 발생하며, 준비된 에이전트가 작동하여 TIL 형식으로 변환된 코드를 GitHub에 푸시합니다. 이후 README.md 파일에 해당 인덱스가 추가되고 마지막으로 완료 메시지를 발신합니다.

### Core
1. **디스코드 메시지 트리거**
   * n8n에서 Discord 노드를 사용하여 새로운 메시지 수신 시 워크플로우가 시작됩니다.

2. **GitHub 푸시 및 README 업데이트**
   * 변환된 코드가 GitHub 저장소에 푸시됩니다.
   * 이후 자동으로 README.md 파일에 인덱스가 업데이트됩니다. 
   ```javascript
   // 예제 코드
   const fs = require('fs');
   const content = '\n- 신규 항목 추가';
   fs.appendFileSync('README.md', content);
   ```

3. **완료 메시지 발신**
   * 모든 과정이 완료되면 성공 메시지를 디스코드로 발신합니다.

### Insight
n8n을 처음 사용해보았는데, 기본적인 것은 두 가지만 알면 어렵지 않다는 것을 알았습니다. 노드에는 어떤 입력과 출력을 가지는지 정확히 이해하는 것이 워크플로우 설계의 핵심입니다. 이 어려운 부분을 간단하게 이해하게 해준 존재에게 감사하는 마음입니다.
